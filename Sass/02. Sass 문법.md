# 기본 문법

## 변수

scope는 블록이다. (!default 및 !global의 경우 필요하면 공부하자.)

```SCSS
$font-stack:    Helvetica, sans-serif;
$primary-color: #333;

body {
  font: 100% $font-stack;
  color: $primary-color;
}
```

## nesting

기본적인 nesting 및 >, +, ~는 &를 필요로 하지 않는다.

```SCSS
nav {

  a {
    display: block;
    padding: 6px 12px;
    text-decoration: none;
  }

  > div {
    display: block;
  }
  + div {}
  ~ div{}

}
```

## & - 상위 선택자 참조

[자세한 가이드](https://css-tricks.com/the-sass-ampersand/)

```SCSS
div{
  // pseudo class
  &:hover {
    color: #000;
  }
  //adding class
  &.adding-class{

  }
}
```

## #{} - interpolation

```SCSS
$name: foo;
$attr: border;

div.#{$name} {            // p.foo
  #{$attr}-color: blue; // border-color: blue;
}

```

## 속성 이름 묶기

```SCSS
.div {
  font: {
    weight: 700;
    size: 10px;
    family: sans-serif;
  }
}
```

```SCSS
.div {
  font-weight: 700;
  font-size: 10px;
  font-family: sans-serif;
}
```

# 재사용

## use

다른 코드를 Scss 확장자를 제외한 `@use '<코드 이름>';`으로 삽입할 수 있고, 변수의 경우 `<use한 코드명>.$<변수명>`으로, 아래에서 배울 mixin의 경우 `@include <use한 코드명>.<mixin명>`으로 삽입 가능하다.

```SCSS
// _base.scss
$some-color: hsl(0, 100%, 50%);

body {
  color: gray;
}

```

```SCSS
// main.scss
@use 'base';

div {
  color: base.$some-color;
}
```

결과는 다음과 같다.

```CSS
/* CSS */
body {
  color: gray;
}

div {
  color: hsl(0, 100%, 50%);
}
```

## %와 extend

placeholder인 %의 경우 CSS에서 보이지 않는다. 물론 일반적인 클래스에 대해서도 extend는 사용할 수 있다.

```SCSS
%message-shared {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.message {
  @extend %message-shared;
}

.success {
  @extend %message-shared;
  border-color: green;
}

.error {
  @extend %message-shared;
  border-color: red;
}
```

결과는 다음과 같다.

```CSS
/* CSS */
.message, .success, .error {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  border-color: green;
}

.error {
  border-color: red;
}
```

## mixin과 include

```SCSS
@mixin theme($theme: DarkGray) {
  background: $theme;
  box-shadow: 0 0 1px rgba($theme, .25);
  color: #fff;
}

.info {
  @include theme;
}
.alert {
  @include theme($theme: DarkRed);
}
.success {
  @include theme($theme: DarkGreen);
}
```

결과는 다음과 같다.

```CSS
/* CSS */
.info {
  background: DarkGray;
  box-shadow: 0 0 1px rgba(169, 169, 169, 0.25);
  color: #fff;
}

.alert {
  background: DarkRed;
  box-shadow: 0 0 1px rgba(139, 0, 0, 0.25);
  color: #fff;
}

.success {
  background: DarkGreen;
  box-shadow: 0 0 1px rgba(0, 100, 0, 0.25);
  color: #fff;
}
```

## mixin와 extend 중 무엇을 언제 사용하는가?

둘 모두 다른 코드를 가져온다는 점에서 같으며 HTML에 적용되는 결과물도 같다. **차이점은 CSS 코드에서 나온다**. mixin의 경우 각 선택자에 대해 declaration(선언)을 추가하고, extend의 경우 그것들을 하나로 모아 선언을 추가한다.

따라서 mixin은 연관성이 없는 코드들 간에 재사용이 필요할 때 활용하고, extend는 연관성이 있는 코드들 간에 재사용이 필요할 때 활용하면 된다.

**그러나 대부분의 상황에서 mixin을 권장한다.**

```CSS
/* extend CSS */
.message, .success, .error {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  border-color: green;
}

.error {
  border-color: red;
}
```

```CSS
/* mixin CSS */
.message {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
  border-color: green;
}

.error {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
  border-color: red;
}
```

# 흐름제어

## if문

else가 있지 않은 한 if 앞뒤로는 빈 새줄이 있는 것이 좋다.

```SCSS
@if not ($value == 15) {
  // …
} @else {
  // …
}
```

## each

each의 경우 리스트와 맵을 순회하는 데 자주 쓰인다. index() 내장 함수를 활용할 수도 있다.

```SCSS
@each $theme in $themes {
  .section-#{$theme} {
    background-color: map-get($colors, $theme);
  }
}
```

```SCSS
@each $key, $value in $map {
  .section-#{$key} {
    background-color: $value;
  }
}
```

## for

for의 경우 i를 통해 nth 클래스와 결합해 사용할 수 있다.

```SCSS
@for $i from 1 through 10 {
  .div:nth-of-type(#{$i}) {
    border-color: hsl($i * 36, 50%, 50%);
  }
}
```
